Simulation of Elevator Group in Buildings

Up&Down – About application

Application itself allows to configure common group of elevators for building and list with statistical data about visitors of building via graphical user interface. Output of application is report containing statistical data which represents load of the system, this data helps user to assess suitability of created configuration

Model configuration

User must set up two parts of model – lift system via Model Configurator Window (figure 1) and list of visitors.

Model Configurator Window

Figure 1: Model Configuration Window

Configuration is consist of three steps – base parameters of building (blue section), lifts configuration (purple section) and travellers configuration (red section).

Step 1 - base parameters

  • Numbef of lifts – Integer, greater than 0.
  • Numbef of floors – Integer, greater than 1.

Step 2 - Lifts parameters

  • Base floor – Vertical position of lift. Integer, greater than 0 and less than final floor.
  • Final floor – Vertical position of lift. Integer, greater than base floor and less or equal to number of floors.
  • List of forbidden floors – Floors where lift never stop. String or Scalar/vector of integers eg. none; 3; 2 4 6; Greater than base floor and less than final floor.
  • Capacity – Integer.
  • Speed – Decimal, from 0.1 to 0.7.
Each lift must be set via "Set one lift" button. After setting up all of lifts, button "Confirm lifts configuration" set up model and user can view model visualization via "Show plan of building".

Step 3 - System and travellers configuration

User chooses the behavior of the system and lists the passenger list.

List of visitors

List is simple text file, where each line represent one traveller and one line is consist of three elements seperated by comma.

iteration of arrival, desired floor, start floor
eg:
10,5,1
181,3,7

User must to configurate this file by his own or use simple generator in Step 3 (Model Configuration Window).

Simulation results

User can view previous reports about simulations that are stored in the output folder.
